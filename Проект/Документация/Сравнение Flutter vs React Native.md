# Flutter vs React Native — анализ для Habit Tracker

## Контекст проекта

- Habit Tracker с детальной кастомизацией UI
- iOS + Web (браузер)
- Backend: Java (Spring Boot)
- Синхронизация данных через REST API

---

## 1. Кастомизация UI

| Аспект | React Native | Flutter |
|--------|-------------|---------|
| Кастомные компоненты | Через нативные компоненты + стилизация | Полностью свой рендеринг (Skia) — рисует всё сам |
| Анимации | Reanimated — мощный, но сложнее | Встроенные, плавные, проще API |
| Кастомные графики/чарты | Библиотеки (Victory, react-native-chart-kit) | fl_chart, custom paint — рисуй что хочешь |
| Уникальный дизайн | Возможен, но борьба с нативными компонентами | Идеален — ты контролируешь каждый пиксель |

**Verdict:** Для "детальной кастомизации" **Flutter лучше**. Он рисует UI сам, не зависит от нативных компонентов платформы.

---

## 2. Web-версия

| Аспект | React Native (Expo) | Flutter Web |
|--------|-------------------|-------------|
| Зрелость | React Native Web — зрелый, по сути обычный React | Стабильный, но уступает нативному вебу |
| SEO | Хороший (DOM-рендеринг) | Плохой (Canvas/CanvasKit) |
| Размер бандла | ~200–400 KB | ~1.5–2 MB (CanvasKit) |
| Ощущение в браузере | Как обычный сайт | Как встроенное приложение (не совсем "веб") |
| Адаптивность | Flexbox, знакомый подход | MediaQuery, LayoutBuilder |

**Verdict:** Для качественной веб-версии **React Native лучше**. Flutter Web работает, но тяжелее и менее "вебовый".

| Фича | React Native | Flutter |

## Рекомендация

### Выбирай **Flutter**, если:
- Кастомизация UI — приоритет (а для Habit Tracker это так)
- Веб-версия нужна как "приложение в браузере" (dashboard-стиль), а не как полноценный сайт
- Хочешь одну кодовую базу с минимумом платформенных различий

### Выбирай **React Native**, если:
- Уже знаешь React/TypeScript
- Веб-версия должна выглядеть и работать как обычный сайт
- Важна максимальная экосистема npm-пакетов

### Для Habit Tracker я рекомендую → **Flutter**

Причины:
1. **Кастомизация** — главная фича проекта, Flutter здесь сильнее
2. **Один Dart-код** для iOS и Web — меньше дублирования
3. **Анимации** streak'ов, прогресс-баров, графиков — Flutter делает это нативно
4. Веб-версия Habit Tracker — это dashboard, а не контентный сайт, Flutter Web для этого подходит
