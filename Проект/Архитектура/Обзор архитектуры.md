### Таблицы

| Таблица | Назначение |
|---------|-----------|
| `users` | Пользователи, авторизация |
| `habits` | Привычки с кастомизацией (цвет, иконка, идентичность, мини-версия) |
| `habit_completions` | Отметки выполнения (дата + привычка) |
| `habit_chains` | Цепочки привычек (Habit Stacking) |
| `habit_chain_items` | Привычки в цепочке с порядком |
| `freeze_days` | Замороженные дни (Freeze Day) |
| `achievements` | Достижения и milestone'ы пользователя |

### Ключевые индексы
- `habit_completions (habit_id, completed_date)` — UNIQUE, быстрый поиск отметок
- `habits (user_id, is_archived)` — список активных привычек
- `habit_completions (habit_id, completed_date DESC)` — расчёт streak'а
- `freeze_days (user_id, freeze_date)` — UNIQUE

## Взаимодействие компонентов

### Поток: отметка привычки
```
1. Пользователь нажимает галочку (один тап)
2. Frontend:
   - Мгновенно показывает анимацию (optimistic update)
   - Обновляет прогресс-бар дня
   - Отправляет POST /api/habits/{id}/complete
3. Backend:
   - Проверяет JWT
   - Сохраняет HabitCompletion
   - Пересчитывает streak
   - Проверяет milestone (7, 21, 30... дней)
   - Если milestone → создаёт Achievement
4. Frontend:
   - Получает ответ с обновлённым streak'ом
   - Если все привычки за день → анимация "Perfect Day!"
   - Если milestone → показывает достижение
```

### Поток: расчёт streak'а
```
1. Получить все completions для привычки, отсортированные по дате DESC
2. Получить freeze_days для привычки
3. Идти от сегодня назад:
   - Если дата есть в completions → streak +1
   - Если дата есть в freeze_days → streak +1 (день заморожен)
   - Если дата пропущена → streak прерван
4. Вернуть: current_streak, best_streak, total_completions
```

## Безопасность

### JWT Flow
```
1. Login → Backend возвращает access_token (15 мин) + refresh_token (30 дней)
2. Каждый запрос → Authorization: Bearer {access_token}
3. Если access_token истёк → Frontend отправляет refresh_token
4. Backend проверяет refresh_token → выдаёт новую пару токенов
5. Если refresh_token истёк → пользователь логинится заново
```

### CORS
- Разрешённые origins: домен Vercel (web) + localhost (dev)
- Разрешённые методы: GET, POST, PUT, PATCH, DELETE
